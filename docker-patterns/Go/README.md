# Go

## 目的

Go アプリケーション向けの軽量な実行イメージ構成を整理するパターンです。

## このパターンが向く場面

- 単一バイナリで配布できる Go サービスを小さく保ちたい
- マルチステージビルドと静的リンクの基本を押さえたい

## 主な構成

- `Dockerfile`

## 設計ポイント

- ビルドステージと実行ステージを分離している
- `CGO_ENABLED=0` を前提に静的リンクされたバイナリを生成する
- 実行ステージは軽量イメージを使い、不要ファイルを含めない

## 代替案

- distroless イメージを最終ステージに使う
- Alpine ではなく Debian slim を使って互換性を優先する
- 開発時だけは単一ステージでビルド速度を優先する

## 使い方

1. イメージをビルドする

```bash
docker build -t golang-app .
```

2. コンテナを起動する

```bash
docker run -p 8080:8080 golang-app
```

## 注意点

- アプリケーションは 8080 ポートで待ち受ける前提
- CGO 依存がある場合は Dockerfile のビルド方針を見直す
- 本番では非 root 実行や distroless イメージの検討余地がある
